## Lesson Shape

This chapter extends volume away from direct formula use and into cube-model reasoning:

- count a 3-D cube model by slicing it into layers
- mark missing cubes in each layer before subtracting
- use multiple layer plans to find the total volume
- count adjacent cube/tower models where touching faces are hidden but cubes still count
- handle tunnels and hidden missing cubes without double-counting overlaps
- read plan-view numbers as tower heights
- count staircase solids by rows, layers, or triangular totals
- use front, left, and plan views together
- read a front view or left view directly from a tower plan
- work backwards from final volume to missing cubes per layer
- connect counted cube solids to water-displacement extension questions

## Named Module Classics

1. **Layer Count** - count one layer, then multiply by the number of identical layers.
2. **Layer Map Sum** - add separate layer-plan counts.
3. **Adjacent Cubes** - count connected towers/cubes even when adjacent faces are hidden.
4. **Tunnel Overlap** - subtract tunnels while correcting overlap double-counting.
5. **Side Slice Tunnel** - subtract a repeated missing rectangular slice from a cuboid.
6. **Height Map** - add plan-view tower heights.
7. **Stair Stack** - use `1 + 2 + ... + n` in a stepped cross-section, then stretch by length.
8. **Front & Left Views** - read the tallest tower in each column or row to form a view.
9. **View Rebuild** - use front, left, and plan views to check/rebuild the solid.
10. **Reverse Missing** - divide the total by layers and compare with a full layer.
11. **Cube Displacement** - counted unit cubes become cubic-centimetre volume in a tank.
12. **Compare Solids** - compare layer-count totals rather than visual size.

## Diagram Fidelity Requirements

- Layer diagrams must show individual grids with missing cells marked.
- Adjacent-cube diagrams must show touching cells/cubes and state that hidden faces do not reduce volume.
- Tunnel diagrams must label the overlap or tunnel opening so children do not subtract blindly.
- Height maps must make it clear that a number inside a square is a tower height, not a label for the square.
- View problems must include both direct front/left-view reading and full plan/front/left rebuild questions.
- Front-view labels must be tied to columns; left-view labels must be tied to rows.
- Water-extension diagrams must name the object volume and tank base area separately.

## Variation Rule

The app preserves the source problem structures but uses fresh generated numbers and grids. This prevents children from answering by memorising page diagrams while keeping the lesson logic intact.
