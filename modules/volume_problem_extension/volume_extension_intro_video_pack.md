# Volume Problem Extension Intro Video Pack

Module: Volume Problem Extension

Source: `Y6-M-L10-L18-Study Book.pdf`, Lesson 16, PDF pages 109-125.

Audience: 11-12 year olds preparing for the Volume Problem Extension practice round.

Length target: 150-180 seconds.

## Learning Outcome

After the intro, the learner should be able to turn a 3-D cube model into countable evidence: layer maps, removed cells, adjacent cubes, tunnels with overlap, height maps, front-and-left views, view rebuilds, reverse counting, and cube displacement. Every move starts from a picture before any arithmetic, so the visual reason is locked in before the formula.

## Named Skills

The learner sees this skill card before any scene starts. Each skill maps to a registry token used by the practice bank:

- Layer maps - slice the solid into floors and add the kept squares in each layer.
- Tunnel overlap - subtract crossing tunnels through a cube without double-counting the shared cube.
- Height maps - read plan-view numbers as tower heights and add them up.
- Views - combine front, left, and plan views to rebuild and count a cube solid.
- Cube displacement - convert a counted cube solid into displaced volume for water-rise problems.

## Scene Storyboard

| Scene | Title | Purpose | Child-facing voiceover |
|---|---|---|---|
| 1 | Slice The Solid | Show layer maps before counting. | "A hard 3-D shape becomes easier when we slice it into floors. Count each floor, then add the floors. That gives us the layer maps approach." |
| 2 | Mark Missing Cubes | Make removed blocks visible. | "If a square is missing in a layer, mark it first. A 3 by 3 layer with one missing square has 9 minus 1, so 8 cubes left in that layer." |
| 3 | Tunnels Can Overlap | Prevent tunnel overlap double-subtraction. | "When two tunnels cross, the middle cube belongs to both tunnels. Subtract the tunnels first, then add the overlap back once so we do not lose that cube twice." |
| 4 | Adjacent Cubes Still Count | Stop visible-face counting. | "When cube towers touch side by side, some faces disappear from view. Those hidden cubes are still real, so we must add every tower in the layer." |
| 5 | Read Height Maps | Teach plan numbers as towers. | "A number in a top-view square tells us how many cubes are stacked there. Add the numbers across the height map to get the volume." |
| 6 | Stairs Are Layers | Count stepped solids cleanly. | "A staircase is 1, then 2, then 3 cubes in a slice. Use the triangular total, then stretch that count through the length of the staircase." |
| 7 | Views Are Clues | Combine front, left, and plan views. | "One view is not enough. The front view gives column heights, the left view gives row heights, and the plan view shows where cubes are allowed to sit." |
| 8 | View Direction Matters | Practise front and left views directly. | "For the front view, take the tallest tower in each column. For the left view, take the tallest tower in each row. The direction changes the answer." |
| 9 | Reverse The Count | Work backwards from a known volume. | "If the total volume is given, divide by the number of layers first to find cubes per layer, then compare with the full layer to find the missing cubes." |
| 10 | Cubes Move Water | Link cube displacement to water rise. | "If each unit cube is one cubic centimetre, the cube count tells us the volume that pushes water up. That volume divided by base area gives the water rise." |
| 11 | Ready To Rebuild | Map intro skills to practice. | "Now train with fresh adjacent cubes, layer maps, tunnels, height maps, views, stairs, and water extensions. Name the move before you write a number." |

## Warmup Gate

1. Count a 3 by 3 layer with one missing square (`8` cubes).
2. Add three layer counts from a small layer map.
3. Find the overlap correction for two crossing tunnels.
4. Add a small adjacent-cube plan with hidden faces.
5. Read one front view and one left view from a tower plan.

## Misconception Risks

- Counting visible cubes only and forgetting the hidden adjacent cubes.
- Subtracting crossing tunnels twice at their shared cube instead of adding the overlap back.
- Losing cubes because adjacent faces are hidden between two towers.
- Reading plan-view height numbers as decorative labels instead of tower heights.
- Mixing up front-view columns with left-view rows because the page is rotated.
- Trusting one view without checking the plan view for allowed positions.
- Using total tank height instead of the displaced volume divided by base area in cube displacement questions.

## Voiceover Draft

A 3-D cube solid is hard to count from one angle. The fix is to slice it into floors and read each floor as a small layer map. Count what is kept in each floor, mark the missing squares with a cross, then add the floors. That single idea covers most of the chapter.

When the source draws a tunnel through a cube, two questions are hiding. First, how many cubes does the tunnel remove. Second, do any tunnels cross. If they do, the overlap cube belongs to both tunnels, so subtracting both tunnels removes that cube twice. The fix is to add the overlap back once.

When two towers of cubes sit next to each other, the touching faces disappear from view. The cubes are still real, so we add every tower in the layer. The drawing hides faces, but it never hides volume.

Height maps replace the layer-map idea with a single plan view labelled by tower heights. A square labelled `3` means a tower of three cubes sits there. Add all the labels to get the volume. Stairs are the same idea in a tidy shape: one row, then two rows, then three rows, stretched through the length.

Views are the third tool. The front view shows the tallest tower in each column. The left view shows the tallest tower in each row. The plan view shows where cubes can sit. Combine all three to rebuild and count the solid, especially when the drawing is small or hidden cubes are likely.

When the question gives a final volume, work backwards. Divide by the number of layers to find cubes per layer, then compare with the full layer to find the missing cubes. Reverse counting is the same skill in reverse direction.

Cube displacement links this whole chapter back to water. If each cube measures one centimetre on each side, the cube count is the displaced volume in cubic centimetres. That volume divided by the tank base area gives the water rise when the solid goes under.

By the end of this intro, every practice problem fits one of the five named moves: layer maps, tunnel overlap, height maps, views, or cube displacement.

## NotebookLM Steering Prompt

Create a short explainer for an 11-year-old learning Volume Problem Extension. Use the supplied Lesson 16 pages. Start with the idea that a difficult cube solid can be sliced into layer maps. Teach how to mark missing cells, count each layer, add layer plans, count adjacent cube towers even when faces are hidden, and handle tunnels that overlap. Then introduce plan-view height maps, staircase solids, direct front-view and left-view reading, and front-left-plan view reconstruction. End by connecting counted unit cubes to cubic-centimetre volume for cube displacement. Keep every example visual, concrete, and based on fresh numbers rather than memorised page diagrams.
